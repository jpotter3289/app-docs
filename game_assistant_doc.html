Game Assistant App

This is just a test line to tell me a modification has been done

The basic purpose of this app is to hold collectively all existing types of dice and a coin flipping mechanic. Users playing cards, D&D role-players, and other role-playing gamers will no longer need to carry boxes of dice around to determine rolls or 50/50 chances.

Main Page:

This page shows the user what they can roll, an option on how many rolls they can make at once, an actual roll button, and a final button to view the results of those rolls.

Header:

Simple title header

Listeners:

  • listens to the 'Navigate here' channel

Output:

  • no output needed

Image Menu:

User can navigate between multiple images, these visually represent the coin and dice types.

Listeners:

  • listens to the users button click which will cycle through the images.

Output:

  • outputs a label based on the current image shown, this label gets sent to the dice component

Counter:

User interacts with this to change the number of rolls to make at once.

Listeners:

  • listens to a channel to change the number.

Output:

  • sends out an integer value. Default is 1.

Button:

Named "Roll" or "Flip" depending on which image is currently on the image menu; This button sends a signal to the dice component to generate a number.

Listeners:

  • listens to a channel from the image menu to change the text to say "flip" or "roll".

Output:

  • sends out a channel to the "give random number" listener of the random number component

Button:

Named "See Results Page".This button sends a signal to the results page header to change pages for the user.

Listeners:

  • listens to a channel from the results page that tells it that there is a list made, if no list has been made this button is inactive

Output:

  • sends out a signal to the "Results Page" header to change pages

Results Page:

This page is where the user can view the dice roll results. Clear the list, and go back to the roll page without clearing the list.

Header:

Simple title header to tell the user they are on the results page.

Listeners:

  • listens for a button callout from the main page.

Output:

  • no ouput channel

List Component:

Shows a list of strings or integers. More specifically it shows the list of rolls made on the main page.

Listeners:

  • listens to the text format component and adds a new line to the list
  • listens to a channel to delete the last input made.
  • listens to a channel to clear the entire list.

Output:

  • sends out a signal to the "see results button" to tell it that there is a list and the button can now be clicked

Button:

Named "Go back"; Brings the user to the toss page without clearing the list component.

Listeners:

  • no listener channel needed for this app

Output:

  • Puts out a channel to the header on the main page to change pages.

Button:

Named "Start Over"; clears the list component and brings the user back to the first page

Listeners:

  • listens to the list component to tell it if there is a list to clear, if not this button is inactive.

Output:

  • sends a channel out to the list component to tell it to clear the data.
  • sends out a channel to the header on the main page to change pages.

Components Page:

This page is where the dice component and text formatter components exist. The info given in the first page are sent to these components which then send their own info to the results page.Users will not see this page.(Author's backend)

Dice Component:

This fires off one or more randomly generated numbers

Listeners:

  • listens for a channel from the image menu for that current images label to tell it which type of roll to make.
  • listens to a channel from the counter component to tell it how many rolls it should make at once.
  • listens to a channel from a button on the main page to signal that it can now generate a random number.

Output:

  • outputs the randomly generated number to the list component. (Default is set to a coin flip; min-1 max-2)
  • outputs a number generated numbers to the list component. (Default amount of numbers outputted is 1)

Text Formatter:

This component takes integer information which it receives through listener channels and outputs strings to suit the authors app needs. This is useful for parsing data into pre-made sentences. For this apps purpose; for every roll fired it makes a new line taking the label and the generated value from the dice component, then places them into a sentence structure like "Rolled a (D6): Got (5)".

Listeners:

  • listens for the integer value from the dice component and places it into a pre-made sentence structure.
  • listens to the label connected to given value and places it into a pre-made sentence structure
  • listens to a confirm string signal that will tell it to ouput the finished sentence structure

Output:

  • ouputs a signal to a counter component to check if the string has received all the data it needs to ouput it
  • outputs a finished string that contains the given data that can be read by the list component.
  • outputs a number of generated strings to the list component. (Default amount of numbers outputted is 1)

Components Needed:

  • Image Menu
  • Interactable Counter
  • Text Formatter
  • List Component
  • Audio Component
